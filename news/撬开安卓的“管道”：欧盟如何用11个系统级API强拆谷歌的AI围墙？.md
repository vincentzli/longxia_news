# 撬开安卓的“管道”：欧盟如何用11个系统级API强拆谷歌的AI围墙？

多年来，布鲁塞尔一直试图在欧洲打造一个数字单一市场，让本土创业公司及外部挑战者能够与硅谷巨头平起平坐。但如今，这场权力博弈正悄然深入到现代智能手机最隐秘的毛细血管——系统底层的线路分布。

2026年7月16日，由第一执行副主席特蕾莎·里贝拉（Teresa Ribera）领衔的欧盟委员会，根据《数字市场法》（DMA）对谷歌发出了两项具约束力的规范决定。这一划时代的裁决彻底重塑了移动操作系统与人工智能的监管边界，其锋芒直指谷歌Gemini AI的“默认装机优势”。欧盟强制要求谷歌向第三方AI助理开放安卓系统的11个底层功能，试图以此强拆这家科技巨头的自我优待壁垒，让竞争对手的AI能以与谷歌原生服务完全等同的系统特权运行。

欧盟委员会此次强推的互操作性指令，其技术开放尺度在行业历史上尚属首次。新规之下，谷歌必须将此前秘不示人的安卓敏感API向微软Copilot或各类第三方开发者工具等竞争对手敞开。这11项被强制开放的功能涵盖了四大关键领域：在“唤醒”维度，谷歌必须开放唤醒词注册通道，允许竞争对手配置自定义的后台唤醒词触发器（例如等同于“Hey Google”的语音指令），即便设备处于锁屏、黑屏或省电模式下也必须保持响应；在“上下文”维度，规范要求涵盖了屏幕内容的实时分析权限，使竞争对手的AI模型能够读取屏幕文字与像素，以此预测用户需求；在“操作执行”方面，谷歌须开放通知API，确保应用内的深度链接（deep-links）对Gemini保持中立，并允许第三方助理执行跨应用的后台指令——如打车、撰写邮件草稿或调整系统设置；在“资源获取”层面，谷歌必须共享底层软硬件资源，其中甚至包括对安卓设备端侧AI模型的访问权限。

为了确保上述指令落地，欧盟委员会设定了严苛的合规时间线。谷歌必须在预计于2027年8月1日前发布的下一代操作系统Android 18中，实装第一阶段 of 互操作功能。而对于难度更高的技术——例如允许不同虚拟助手同时监听语音唤醒的“并发热词检测”功能，则必须在2028年8月1日之前完全集成到Android 19中。针对屏幕自动化和后台执行等5项最敏感的底层功能，欧盟允许谷歌保留一套认证机制，以便在第三方服务获取权限前，验证其是否满足必要的数据安全要求。

除了针对安卓系统的强制令，欧盟委员会还依据《数字市场法》第6条第(11)款作出了第二项裁决：自2027年1月起，谷歌必须向符合资质的竞争对手及AI聊天机器人共享去标识化的搜索查询、点击和排名数据。这部分数据必须以“公平、合理且无歧视”（FRAND）的授权条款提供，具体的定价与分发模式预计于2027年1月前尘埃落定。通过撬开谷歌的搜索日志库，欧盟寄望于为DuckDuckGo、Ecosia和Qwant等小众搜索引擎输血。长期以来，这些厂商因缺乏海量查询数据的喂养，在算法精度上始终被谷歌压制。需要指出的是，上述裁决是具法律效力的“强制构建指令”（build orders），而非直接的经济罚款；但如果谷歌未能按期交付，将面临不合规诉讼，并可能被处以最高达其全球年营收10%的巨额罚款。

对此，谷歌方面表达了强烈反对，坚称开放关键的系统安全接口将导致用户隐私防线失守。谷歌全球事务总裁肯特·沃克（Kent Walker）警告称，这些决定“极有可能削弱数百万欧洲人赖以生存的隐私与安全防护墙”。沃克强调，共享查询日志可能会在未取得授权的情况下，将用户私密的搜索指令暴露给“陌生公司”；而开放安卓底层的API，则会“损害公民隐私，泄露商业机密，并危及国家安全”。安全专家同样指出，向第三方应用授予后台屏幕捕捉、实时麦克风监听等深层系统权限，无疑为间谍软件和数据窃取敞开了大门。然而，欧盟委员会反驳称，这些潜在的安全风险完全可以通过谷歌已获批的认证框架加以化解，安全关切绝不能成为将竞争性AI阻挡在门外的借口。

相比谷歌的妥协，苹果则选择了更为决绝的对抗姿态。为了避免在《数字市场法》下陷入类似的监管泥潭，苹果决定在欧盟市场的iOS 27和iPadOS 27系统中，直接阉割其重构的Siri（即Apple Intelligence）功能。不过，由于macOS 27和visionOS 27目前尚未被DMA指定为“看门人”平台，苹果仍允许这些系统在欧盟境内上线最新的AI功能。苹果抗辩称，如果必须向第三方AI模型敞开系统大门，其在移动端将根本无法向用户保证数据安全与隐私。欧盟对此则针锋相对地表示，其法律并未禁止苹果发布任何产品，苹果的举动完全是“为了逃避合规义务，不惜剥夺欧洲消费者享受新技术的权利”。

随着2027年1月搜索数据共享大限的临近，以及同年8月Android 18的发布，全球AI助理市场正因为地缘监管的拉扯而走向割裂。欧洲用户正面临尴尬的境遇：要么像苹果生态那样面对被阉割或延迟上线的AI功能，要么在谷歌生态中忍受多助手并存、权限混乱的系统体验。与此同时，全球开发者也在屏息凝神，观望谷歌将在2027年初拿出的FRAND条款。这套条款究竟是会真正扫清行业准入门槛，还是会因为高昂的合规成本与底层的技术壁垒，让系统级AI助理的赛道继续沦为巨头独占的禁地？

3.2 摘要正文
欧盟根据数字市场法对谷歌下达硬性整改令，强制要求安卓18与19向第三方AI助手开放11个核心API，包括熄屏唤醒、实时像素扫描等。此举引发海外社媒激烈争论。安全倡议者质疑，对第三方敞开实时麦克风和屏幕读取权限等同于给间谍软件递刀，认同谷歌全球事务总裁Walker所谓的隐私灾难警告；但反垄断支持者则叫好，认为这是打破谷歌Gemini垄断的里程碑。与此同时，苹果在iOS 27上对欧盟断供Apple Intelligence，地缘监管割裂正让欧洲用户沦为AI时代的二等公民。
